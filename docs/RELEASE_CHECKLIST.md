# Release Checklist

Use this checklist before publishing a public release.

## Environment

- OS: Windows 10/11 x64
- SDK: .NET SDK 9.0
- Runtime target: `win-x64`
- Build mode: `Release`
- Packaging: self-contained single-file executable

## Local Validation

```powershell
dotnet restore ClickLockNotifier.sln
dotnet build ClickLockNotifier.sln
.\scripts\publish.ps1
```

Expected release output:

```text
dist/win-x64/ClickLockNotifier.exe
```

## Upload Risk Check

Before uploading:

- Do not upload `bin/`, `obj/`, `.vs/`, temporary files, logs, or local clones.
- Do not upload personal absolute paths, download folders, or local machine-specific files.
- Do not upload GitHub tokens, environment dumps, browser profiles, or credentials.
- Keep sound license notes in `THIRD_PARTY_NOTICES.md`.
- Publish binary builds as GitHub Release assets, not as normal source files.
- Confirm the final archive contains only source, docs, assets intended for redistribution, and release binaries.

## Known Behavioral Risks

- Raw Input observes physical mouse button events.
- Windows Mouse ClickLock can alter normal mouse message ordering.
- Fullscreen detection is based on the foreground window rectangle and monitor bounds.
- Some games may disallow input helpers even if they do not modify game memory.

## Suggested GitHub Release Assets

- `ClickLockNotifier-win-x64.zip`
- Source code archive generated by GitHub
